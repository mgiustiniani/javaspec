# PHPSpec semantic compatibility matrix

This matrix is the release-facing companion to [`docs/phpspec-compatibility-charter.md`](phpspec-compatibility-charter.md). It treats PHPSpec-first semantics as the product core of javaspec, not as an optional parity layer.

Status vocabulary:

- `COMPATIBLE`: observable behavior is coherent with PHPSpec.
- `JAVA_ADAPTED`: the same principle is expressed in idiomatic Java.
- `INTENTIONALLY_DIFFERENT`: the difference is deliberate, motivated, documented, and tested.
- `PARTIAL_BLOCKING_1_0`: partial support would make a core 1.0 promise incoherent.
- `PARTIAL_POST_1_0`: 1.0 remains coherent, but parity can expand later.
- `NOT_APPLICABLE_TO_JAVA`: no sensible Java equivalent.
- `UNSPECIFIED`: temporary state; not acceptable at API freeze.

## Mandatory 1.0 semantic acceptance scenarios

These scenarios must be represented by automated tests or explicitly mapped to existing tests before RC:

1. Subject simple: missing subject -> RED or generation plan -> minimal skeleton -> RED until domain logic -> GREEN.
2. Custom construction: constructor/factory selection, overload ambiguity diagnostics, class/record semantics.
3. Collaborator and prediction: `let` configures collaborator, subject receives same collaborator, prediction checked, `letGo` always runs with correct precedence.
4. Matcher and diagnostics: expectation failure is FAILED with expected/actual/source context where available, not BROKEN.
5. Example data: row order, lifecycle, failing-row identity, selection semantics, JSON/XML/JUnit Platform translation.
6. Pending and generation: pending/stub/generated-incomplete behavior cannot be presented as final GREEN.
7. Adapters: CLI, Maven, Gradle, and JUnit Platform preserve the same javaspec semantics without requiring Jupiter authoring.

## First semantic P0 after matrix creation

`Generated incomplete behavior` and `Safe behavior-driven generation` are the first semantic P0s after this matrix because accidental GREEN would contradict the PHPSpec workflow even if CLI/adapters/release mechanics are correct.
